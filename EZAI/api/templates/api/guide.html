Documentation Guide

Let's get started!

When creating documentation it is easy to forget details that are needed when using the model. Therefore we at ezai believe that it is beneficial for us to help you create the best documentation possible. If you ever feel lost, you can return to these guides in the future.

An initial documentation is automatically created when your model is submitted. You can edit the documentation by finding your model in the “model zoo” section of the page.

Formatting

  • • Overview: A description of your model and what it does. It is important to describe what you intend the model to be used for.


  • • Usage: Explanation about the usage of your model. In this section it is necessary to include code examples in at least one language (the language does not matter).


  • • Data: A description of the data used to train the model. This is one of the most important steps. It is even recommended that you provide a link to the data used.
